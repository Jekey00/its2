Betriebssysteme

1. Was versteht man unter einem Betriebssystem?
ASoftware, die Hardware steuert und Anwendungsprogramme unterstützt
BNur ein Texteditor
CEin Hardware-Treiber
DNur eine Benutzeroberfläche
2. Welche Aufgaben übernimmt der Kernel eines Betriebssystems?
AVerwaltung von Speicher, Prozessor, Geräten und Prozessen
BInstallation von Programmen
CDarstellung der grafischen Oberfläche
DNutzerkommunikation im Netzwerk
3. Welches Betriebssystem zählt zu den Echtzeit-Systemen?
AWindows 11
BmacOS Sonoma
CVxWorks oder QNX
DUbuntu Desktop
4. Welche Aussage beschreibt ein eingebettetes Betriebssystem am besten?
AEin OS mit vielen grafischen Effekten
BEin OS, das fest in Geräten wie Waschmaschinen integriert ist
CEin Serverbetriebssystem mit hoher Skalierbarkeit
DEine Cloud-Plattform
5. Worin unterscheidet sich ein Server- von einem Client-Betriebssystem?
AServer-OS stellen Dienste bereit, Client-OS werden direkt von Anwendern genutzt
BServer-OS haben zwingend eine grafische Oberfläche
CClient-OS können keine Netzwerke nutzen
DEs gibt keinen Unterschied
6. Nennen Sie zwei Kriterien, die bei der Auswahl eines Betriebssystems in Unternehmen eine Rolle spielen.
Lösung:
Verwaltbarkeit, Kosten, Software-Kompatibilität, vorhandene Kompetenzen, Homogenität der Systeme, Sicherheit oder Kundenanforderungen.
Lösung ein-/ausblenden
7. Erklären Sie kurz den Begriff Treiber im Zusammenhang mit Betriebssystemen.
Lösung:
Ein Treiber ist ein Softwaremodul, das das Betriebssystem mit einer bestimmten Hardwarekomponente kommunizieren lässt.

Virtualisierung

8. Welche Aussagen treffen auf einen Typ-1 (Bare-Metal) Hypervisor zu? (Mehrfachauswahl)
ALäuft direkt auf der Hardware ohne darunterliegendes Host-OS
BHohe Leistung und häufig in Rechenzentren eingesetzt
CBenötigt zwingend ein Windows-Desktop als Host
DIst immer Open Source
9. Welches Beispiel ist ein Typ-2 (Hosted) Hypervisor?
AVMware ESXi
BKVM (auf Linux-Servern)
COracle VirtualBox auf Windows
DXen (in Citrix Hypervisor)
10. Nennen Sie zwei Vorteile der Virtualisierung. (Mehrfachauswahl)
ABessere Auslastung der Hardware / Kostenersparnis
BIsolation der Systeme (Sicherheit, Testbarkeit)
CUnbegrenzte Performance ohne Overhead
DAutomatische Skalierung ohne Konfiguration
11. Was beschreibt ein Snapshot einer virtuellen Maschine am besten?
AEine permanente Erhöhung der VM-Ressourcen
BEine Sicherung auf Bandlaufwerk
CEine Lizenzform für VMs
DEin Momentaufnahme-Zustand, zu dem die VM zurückgesetzt werden kann
12. Welcher Nachteil ist typisch für Virtualisierung gegenüber Bare-Metal-Installationen?
AVöllig fehlende Isolation
BMehraufwand in Ressourcen (Overhead) und ggf. Performanceverlust
CUnmögliche Migration
DKeine Backups möglich
13. Welche Technologien oder Begriffe gehören direkt zur Virtualisierung? (Mehrfachauswahl)
AHypervisor
BRAID-Level 0
CVirtuelle Maschine (VM)
DHTML
14. Was bedeutet „Live-Migration“ in Virtualisierungsumgebungen?
AVerschieben einer laufenden VM von einem Host zu einem anderen ohne Ausfallzeit
BEine Methode zur Backup-Archivierung
CUpdate des Hypervisors
DEine Cloud-basierte Lizenz
15. Erklären Sie kurz den Unterschied zwischen Virtualisierung und Emulation.
Lösung:
Virtualisierung teilt vorhandene physische Ressourcen und nutzt die gleiche Hardwarearchitektur; Emulation bildet eine andere Hardwarearchitektur nach und kann Performanceeinbußen haben, erlaubt aber die Ausführung andersarchitekturierter Systeme.
Lösung ein-/ausblenden

ALinux

DUnix SVR4

AC:\Users (bzw. Benutzer)
BC:\Program Files (x86)

DC:\Temp

ADas System kann mehr Arbeitsspeicher ansprechen als 32-Bit-Versionen
BNur Programme mit 32 Bit laufen darauf
CDie CPU arbeitet nur mit 32 Bit
DDas Dateisystem wird verschlüsselt

AmacOS basiert auf MS-DOS
BmacOS unterstützt keine grafische Oberfläche
CmacOS ist ein reines Server-Betriebssystem
DmacOS basiert auf Unix und nutzt die Oberfläche „Aqua“

BLinux ist nur für Server verfügbar

21. Welche Linux-Komponenten oder Befehle sind typisch? (Mehrfachauswahl)
AShell / Terminal
BBefehle wie cd, ls, chmod
CRegistry
DFinder
22. Wofür stehen die Kürzel „su“ und „sudo“ unter Linux?
Asu = switch user, sudo = execute command as superuser (Administratorrechte)
Bsu = setup update, sudo = shutdown daemon
CBeide starten automatisch Grafikprogramme
DNur auf macOS vorhanden
23. Nennen Sie zwei Beispiele für Linux-Distributionen und deren typische Einsatzgebiete.
Lösung:
Ubuntu – weit verbreitet im Desktop-Bereich; Debian – Basis für viele Distributionen und Server; Red Hat Enterprise Linux – kommerzielles Serversystem; Kali Linux – Sicherheits- und Pen-Testing.

Aext4
BNTFS
CAPFS
DHFS+
25. Wie lautet das Standard-Dateisystem aktueller Linux-Distributionen?
Aext4
BNTFS
CAPFS
DFAT32
26. In welchem Verzeichnis liegt bei Linux typischerweise der Benutzerordner?
A/root
B/home
C/etc
D/usr/bin
27. Welche Aussage zur Benutzer- und Rechteverwaltung unter Linux ist korrekt?
ARechte werden über Benutzer, Gruppen und Dateiattribute (r, w, x) gesteuert
BEs gibt keine Administratorrechte
CJede Datei gehört immer mehreren Benutzern gleichzeitig
DRechte können nicht geändert werden
28. Erklären Sie kurz die Bedeutung der drei Buchstaben r, w und x bei Linux-Dateirechten.
Lösung:
r = read (lesen), w = write (schreiben), x = execute (ausführen). Sie gelten getrennt für Besitzer, Gruppe und andere Benutzer.

Softwarelizenzen

29. Was beschreibt eine proprietäre Softwarelizenz?
ADie Nutzung ist eingeschränkt, Quellcode ist nicht frei zugänglich
BDie Software darf beliebig verändert und weitergegeben werden
CDie Software gehört der Allgemeinheit
DSie darf ohne Zustimmung verkauft werden
30. Was ist charakteristisch für Open-Source-Software?
ASie darf nicht verändert werden
BSie darf nur privat genutzt werden
CDer Quellcode ist offen und darf unter bestimmten Bedingungen genutzt oder verändert werden
DSie ist grundsätzlich kostenlos und lizenzfrei
31. Welche Organisation definiert die Grundsätze freier Software (Free Software)?
AFree Software Foundation (FSF)
BISO
CWorld Wide Web Consortium (W3C)
DIEEE
32. Nennen Sie zwei Beispiele für bekannte Open-Source-Lizenzen und beschreiben Sie kurz deren Zweck.
Lösung:
GPL (General Public License) – erlaubt Nutzung, Veränderung und Weitergabe, erfordert Offenlegung des Quellcodes.
MIT-Lizenz – sehr frei, erlaubt nahezu beliebige Nutzung mit Hinweis auf Urheber.
Creative Commons – verschiedene Varianten für freie Veröffentlichung von Inhalten.
Lösung ein-/ausblenden
